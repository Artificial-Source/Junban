# Mission and Principles

## Mission

Junban aims to be a local-first task manager that feels simple out of the box, stays private by default, and becomes more capable through optional AI and plugins instead of mandatory accounts or cloud lock-in.

## Product Principles

### 1. Local-first and private by default

- No accounts required for the core product
- No tracking as a default product assumption
- User data stays on the user's machine first

### 2. Useful without cloud dependencies

- Core task management must stand on its own
- AI, sync, and integrations are additive, not required for daily use

### 3. AI-enhanced, not AI-dependent

- AI should help with planning, organization, and automation
- AI failures must not break the core task-management experience

### 4. Extensible without making the core bloated

- Plugins are the preferred path for feature expansion
- The built-in product should stay approachable even as advanced capabilities grow

### 5. Portable data over lock-in

- SQLite is the main path for capability and performance
- Markdown remains an important portability path
- Integrations should preserve user control over data and exits

### 6. Polish the everyday workflow first

- Fast capture, clear prioritization, and dependable scheduling matter more than novelty
- New surfaces should support the core productivity loop, not distract from it
