# Tutorials

Tutorials are for learning Junban by doing. They assume minimal context, move in order, and aim for a concrete result.

## Tutorials in Order

1. [`first-run.md`](first-run.md) for the first local success: install the app, run it, and create a task.
2. [`your-first-plugin.md`](your-first-plugin.md) for the first plugin workflow using the built-in scaffolder and plugin examples.

## Before You Start

- Junban source-run development uses Node.js 22+, `pnpm`, and the scripts in `package.json`.
- The setup flow is grounded in [`../guides/SETUP.md`](../guides/SETUP.md).
- Plugin work builds on the plugin author docs in [`../reference/plugins/API.md`](../reference/plugins/API.md) and [`../reference/plugins/EXAMPLES.md`](../reference/plugins/EXAMPLES.md).

## After the Tutorials

- Use [`../how-to/README.md`](../how-to/README.md) when you have a specific task to solve.
- Use [`../explanation/README.md`](../explanation/README.md) when you want the architectural context behind the workflows.
- Use [`../reference/README.md`](../reference/README.md) for technical lookup.
